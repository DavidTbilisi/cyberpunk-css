CSS Classes & Capabilities

Overview of utility classes, component classes, and SCSS partials used in this project.

SCSS partials

FilePurpose
_variables.scssColor, spacing, and size variables used across components.
_mixins.scssReusable mixins (responsive helpers, gradient helpers, etc.).
_fonts.scssFont-face and typographic scales.
_navbar.scssNavbar styles and scroll-state styles (tied to is-scrolled class).
cyberpunk.scssMain entry importing partials and component rules.

Component classes

ClassMeaning / Usage
.cp-btnPrimary button styling; used with data attributes for ripple and interactions.
.site-headerTop navigation wrapper; receives is-scrolled when page scrolls.
.cp-toastToast element styling (enter/exit transitions controlled by JS).
.cp-modalModal dialog shell; toggled with is-open class.
.hex-tableContainer for hex + ASCII rendering; rendered by hexviewer.js.
.netgraphSVG wrapper class; used by netgraph.js.
.terminalRetro terminal visual styles for demo content and code blocks.
.windowRetro OS window chrome (titlebar, controls, draggable) styles.
.codeCode block styling — monospace, selectable, with subtle glow.

Interaction hooks (CSS + JS)

Some classes are toggled by JS and correspond to visual states. Example:

// Applied by JS when navbar is scrolled
<header class="site-header is-scrolled">…</header>

// Modal open state
<div id="confirm" class="cp-modal is-open">…</div>

Notes & caveats

  • Most colors and sizes are driven from _variables.scss and should be changed there to keep consistency.
  • Styles rely on modern CSS (custom properties, flexbox, grid). For older browser support, run the SCSS through a postcss/autoprefixer step.
  • Visual effects (ripples, packet animations) are implemented with minimal DOM updates; tweak timing in CSS if you adjust JS durations.

Where to start

If you're adding or modifying a component, edit the appropriate partial in the repo root (e.g. _navbar.scss) and rebuild the CSS with the project's sass script:

npm run build:css
