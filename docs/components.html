Components

Buttons

Use the .cp-btn class. Variants via attributes or contextual classes (e.g. data-variant="danger").

<button class="cp-btn">Primary</button>
<button class="cp-btn" data-variant="danger">Danger</button>

Network Topology Graph

Visualize nodes and edges with simple packet animations. This is a simulated view for demos only.

<div class="cp-netgraph" id="demo-netgraph"></div>

Navbar

Markup uses .cp-navbar with link buttons using .cp-navbar__link. Wrap link text in a <span> when you want the typing cursor glyph.

<nav class="cp-navbar">
  <div class="cp-hstack">
    <button class="cp-navbar__link"><span>Home</span></button>
    <button class="cp-navbar__link"><span>Components</span></button>
  </div>
</nav>

Rendered example:

RETROPUNK

Accessibility: links are implemented as buttons by default; ensure they have keyboard focusability and ARIA roles if used as true navigation anchors.

Panels & Cards

Use .cp-panel and .cp-card for content blocks. Panels include grid background mixin for retro feel.

Modals

Structure:

<div class="cp-modal-backdrop" id="demo-modal">
  <div class="cp-modal">...</div>
</div>

Use is-open class on the backdrop to show the modal.

Click to open the modal provided by the demo scripts (openModal() defined in the demo).

Hex / Binary Viewer

Use the .cp-hex component to display file data in offset/hex/ascii columns. The viewer supports copy-to-clipboard.

<div class="cp-hex" id="demo-hex"></div>

Windows (Retro Dialog)

Use .cp-window to render framed dialog windows with a titlebar, controls and footer actions. Good for system-like widgets and panels.

Terminal

The framework includes a terminal component for showing console output or examples. Use .cp-terminal with header, controls and body lines.

nothing to commit, working tree clean
